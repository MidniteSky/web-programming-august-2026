# Learner Guide — Web Programming with JavaScript, HTML & CSS

Welcome! This is a 5-day, hands-on introduction to building for the web. By Friday
afternoon you will have built and **deployed a live, working web application** that
loads and saves data from a server — starting from nothing.

**Before Day 1:** work through the [Setup Guide](setup-guide.md). It takes about
20 minutes and means we can start building, not installing.

---

## The week at a glance

You build **one project all week** — the *Member Manager*, a small membership site —
and it levels up every single day:

| Day | Topic | By the end of the day, your project... |
|---|---|---|
| **1** | How the web works: HTML & first CSS | ...is a real three-page website with structured content and your own colour scheme |
| **2** | CSS in depth: box model, Flexbox, Grid, responsive design | ...looks properly designed, with a real layout — and works on a phone |
| **3** | JavaScript fundamentals & the DOM | ...is interactive: the members table is drawn by your code, with live search |
| **4** | JSON, `fetch` & talking to servers | ...loads and saves real data from a server — add, edit and delete members |
| **5** | Bootstrap, real-world practices & deployment | ...has a professional UI with dialogs and notifications — and is **live on the internet** |

### What each day feels like

Each day runs **09:30–17:00** (online) in four blocks of roughly 80 minutes, with
morning/afternoon breaks and lunch. Every block is the same rhythm:

1. **Short explanation** of a new idea
2. **Live coding together** — the tutor builds a small demo file while you watch/follow
3. **Hands-on exercise** — you build something yourself (this is where the learning happens)
4. **Review** — we look at a couple of solutions together

The day ends with a **capstone session** where you fold the day's new skills into your
Member Manager project.

## How to use the course materials

Each day has its own folder (`day1`, `day2`, ...) that you'll download as a ZIP.
Inside each:

| Folder / file | What it's for |
|---|---|
| `README.md` | That day's outcomes and anything to prepare |
| `demos/` | The small "investigation" files the tutor builds live. **These are yours to break.** Open them, change things, see what happens — they're numbered in teaching order and every one runs standalone. |
| `exercises/` | Your tasks for each block. Each one has a "✅ Done when" line so you know when you're finished. |
| `solutions/` | Worked answers. **Honest advice:** you learn from the struggle, not the solution. Try first, peek second, and if you do peek — *type* the solution out rather than copy-pasting it. |
| `starter/` (Day 2 onwards) | A ready-made copy of the project as it should look at the start of the day. Fell behind yesterday, or your version broke? Start from here, no shame in it — everyone uses it at least once. |
| `my-work/` | *You create this one* — all your own files go here. |

### Ground rules that make the week work

- **Type, don't paste.** Your fingers are learning the syntax as much as your brain is.
- **Ask early.** If you're stuck for more than a couple of minutes, say so in the chat.
  You will never be the only one — you're just the bravest one.
- **Broken code is progress.** Every error message is information. We'll practise
  reading them from Day 1.
- **AI assistants stay off** for the week (the Setup Guide shows how). ChatGPT/Copilot
  can write this code — but *you* can't yet, and that's the entire point of being here.
  There's a note on using AI *after* the course on Day 5.
- **Resist running ahead.** The course introduces things in a deliberate order —
  today's "ugly" page is tomorrow morning's first exercise.

### Reference material we'll use all week

- **MDN Web Docs** — <https://developer.mozilla.org> — *the* reference for HTML, CSS
  and JavaScript. When in doubt all week (and afterwards), search "mdn" + the thing.
- **Your browser's DevTools** (press F12) — we learn a new panel of it every day.
- **[Troubleshooting guide](troubleshooting.md)** — the ten classic "it's broken"
  moments and their fixes. Check it before despairing; it fixes most things.
- **Each day's `CHEATSHEET.md`** — a one-page reference of everything taught that day.
